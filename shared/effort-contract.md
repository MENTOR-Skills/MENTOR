# 工作深度档：行为上限

> 依据设计方案 §2.6
> 本文定义三个深度档位的具体行为边界，被所有功能型技能遵守。

> **现阶段约定（2026-07-17）：基本版本开发期统一按「标准」档执行。** 轻量 / 深入两档的规则已定义（见下文），但暂不向用户开放、技能实现时先不分支，待核心功能稳定后再启用。

---

## 档位总览

| 档位 | 适用场景 | 一句话 |
|------|----------|--------|
| 轻量 | 今晚组会、快速判断一篇论文 | 少检索、短文稿；必须标明局限 |
| 标准（默认） | 日常科研工作 | 完整确认关卡；精读候选需可核对；主张挂证据 |
| 深入 | 开题周、正式相关工作整理 | 更广检索、更长精读、启用挑刺、关系图必出 |

未指定时默认「标准」。用户说「随便写写」时，应建议降为「轻量」并请用户确认。

---

## 各档位行为约束

### 轻量

| 维度 | 上限 |
|------|------|
| 检索范围 | 至多 2 个数据源；检索词不超过 3 组 |
| 精读数量 | 至多 2 篇（快速浏览级别） |
| 输出篇幅 | 全景说明 ≤ 800 字；阅读报告 ≤ 500 字 |
| 确认关卡 | 至少保留：范围确认、付费墙停车 |
| 关系图 | 不生成 |
| 挑刺审阅 | 不启用 |
| 引用核对 | 至少检查 DOI/标题匹配 |
| 必须声明 | 在交付物开头标注「轻量模式，以下分析可能有遗漏」 |

### 标准（默认）

| 维度 | 上限 |
|------|------|
| 检索范围 | 按主题需要，无硬上限；检索词需记录在案 |
| 精读数量 | 精读候选由检索结果产生；用户确认后再精读 |
| 输出篇幅 | 按交付文件自然长度 |
| 确认关卡 | 全部生效 |
| 关系图 | 仅 F2 需要时生成 |
| 挑刺审阅 | 仅 F5 写作时启用（可选） |
| 引用核对 | 全部引用必须经过自动核对 |
| 主张—证据 | 每个关键主张必须有证据来源或标为缺口 |

### 深入

| 维度 | 上限 |
|------|------|
| 检索范围 | 更广数据源（建议 ≥ 3）；补充反向引用检索 |
| 精读数量 | 候选清单需用户确认后逐批精读 |
| 输出篇幅 | 按需要完整展开 |
| 确认关卡 | 全部生效，且高风险主张必须用户逐条确认 |
| 关系图 | 必须生成 |
| 挑刺审阅 | 必须启用（写作产出需要经过挑刺） |
| 引用核对 | 全部引用经过自动核对 + 抽样人工可核验路径 |
| 复现条件 | F3 复现深度附完整检查清单 |

---

## 深度档的流转规则

1. **一次任务只用一个深度档。** 在 `scope.md` 中记录。
2. **后续任务可升级但不可降级。** 例如：轻量浏览后发现关键论文 → 用户可要求对该论文进行标准深读，但原来的轻量全景说明不回溯补充。
3. **跨功能组合时，各功能可独立设定深度档。** 例如：F2 用标准档检索，F3 对其中一篇用深入档。
4. **深度档影响确认关卡的强度，不改变确认关卡的存在性。** 轻量也要停车等确认，只是确认的项目更少。

---

## 降级提示模板

当用户说「随便写写」「快速看看」时，技能应给出：

> 你希望快速完成，我建议使用「轻量」模式：检索更少、报告更短、不画图。输出会标明局限。是否按轻量模式继续？你也可以选择「标准」模式获得完整分析。
