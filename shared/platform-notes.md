# 宿主平台适配说明

> 依据设计方案 §4.3
> 本文记录 MENTOR 技能组在不同平台上的安装、调用与差异。技能正文使用中性表述，平台差异仅在此处维护。

---

## 兼容性总则

| 层 | 策略 |
|----|------|
| 规程与交付约定 | 全宿主共用（技能正文、模板、脚本逻辑） |
| 安装与调用方式 | 写入本文件，技能正文用中性表述 |
| 工具能力下限 | 读文件、写文件、运行终端 Python、访问网络；不把某一家专用插件当默认依赖 |
| 可选能力 | LaTeX 编译、跨模型互审标为可选 |

## Claude Code（当前主要宿主）

### 安装

将本仓库克隆到本地后，在项目根目录创建符号链接指向技能目录：

```powershell
# Windows（需管理员终端；mklink 是 cmd 内置命令，PowerShell 中须经 cmd /c 调用）
cmd /c mklink /D .claude\skills\mentor-skills "<本仓库路径>\skills"
```

> 路径请替换为你机器上的实际仓库位置，不要把个人绝对路径写进技能或脚本。

### 调用方式

用户在对话中描述科研需求，Claude Code 自动匹配技能触发词。常用触发方式：

- 「帮我了解一下 XXX 方向」→ `domain-onboarding`
- 「帮我找 XXX 相关的论文」→ `literature-search-download`
- 「帮我读这篇论文」→ `paper-deep-read`
- 「总结我这周的实验进展」→ `progress-digest`
- 「帮我写组会材料」→ `meeting-brief`

也可以在 `.claude/CLAUDE.md` 中写：
> 科研任务优先遵循 MENTOR 仓库的 AGENTS.md 路由规则。

### 注意事项

- 默认 shell 为 PowerShell，技能脚本使用 `python` 命令（需确保 Python 在 PATH 中）
- 中文文件编码参见 `encoding-utf8.md`
- 如果 `scripts/` 中的 Python 脚本执行失败，检查 PowerShell 执行策略：`Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`

## Cursor

### 安装

1. 打开 Cursor 设置 → Features → Skills（UI 路径以实际版本为准，尚未实测）
2. 添加技能目录：`<本仓库路径>\skills`（替换为实际路径）
3. 或在项目根目录 `.cursor/skills/` 下创建符号链接

### 调用方式

Cursor 根据技能描述中的触发词自动发现并调用。无需额外配置。

### 注意事项

- Cursor 的默认终端可能是 PowerShell 或 Git Bash，脚本中的路径分隔符使用正斜杠
- 如果技能未触发，检查 `SKILL.md` 的 `description` 字段是否包含足够的触发关键词

## 清小搭 / 校内智能共创平台（未来上架）

### 适配策略

| 原组件 | 平台对应 |
|--------|----------|
| 调度模块步骤 | 工作流节点 |
| 确认关卡 | 选择型 / 输入型交互节点 |
| 领域内容包 | 平台文档库 |
| skills/ 中的 Markdown | 平台知识库条目或提示词模板 |

### 注意事项

- 不把 Cursor 专用路径写进平台节点
- 技能正文中的「运行 Python 脚本」表述保持不变，平台侧由节点挂载对应脚本

## 依赖与降级策略

| 依赖 | 基本要求 | 达不到时的降级 |
|------|----------|----------------|
| Python 3.8+ | 文献检索、核对、可视化 | 仅开放「纯模板写作 / 入门问答」子集，并明确声明 |
| 网络（开放文献接口） | F2 检索需要 | 允许用户只提供本地 PDF 进入 F3 深读 |
| LaTeX | 非必须 | 默认 Markdown 交付 |
| 第二模型 | 非必须 | 挑刺技能在同一模型会话内完成 |

## 安装后验证

在任一平台上完成安装后，用以下话术快速验证调度是否正常：

1. 「我对 XXX 方向完全陌生，帮我入门」→ 应进入 F1 工作流
2. 「帮我找 XXX 相关的论文」→ 应进入 F2 工作流
3. 「帮我读这篇论文」→ 应进入 F3 工作流
4. 「总结我最近的实验进展」→ 应进入 F4 工作流
5. 「帮我写组会 PPT 的内容」→ 应进入 F5 工作流

> 注：F4 本周未单派负责人（见 `docs/分工说明.md`），第 4 条在 F4 技能上线前可跳过。
