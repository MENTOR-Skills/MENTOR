# 学术诚信勾选清单

> 依据设计方案 §1.4、§4.1
> 本文件被所有产出型技能（F1–F5）在成稿前强制引用，不得跳过。

---

## 成稿前必查（每次产出报告/全景说明/论文卡片/阶段报告/写作稿前逐条核对）

### A. 文献与引用

- [ ] 文中引用的每一篇论文，都已确认真实存在（DOI 有效、标题匹配、作者匹配）
- [ ] 没有编造引用、页码、DOI、会议名称
- [ ] 模型推断的论文关系（如「A 是 B 的改进」）已显式标注「推断」，未写成既成事实
- [ ] 未将未读论文标注为「已精读」

### B. 实验与数据

- [ ] 没有编造实验数值、曲线趋势、消融结果
- [ ] 没有编造「实验已成功」「已复现」声明
- [ ] 对本地结果的描述有对应日志或输出文件可核验

### C. 下载与获取

- [ ] 没有绕过出版社付费限制获取全文（包括声称「已下载」但实际未获取）
- [ ] 遇到付费墙已停下来请用户补充，而非假装已获取

### D. 隐私与授权

- [ ] 没有在未授权时扫描用户主目录或敏感路径
- [ ] 交付物中不含密钥、令牌、环境变量值、个人隐私数据
- [ ] 未将用户本地材料外传至任何外部服务

### E. 明确声明

- [ ] 对不确定的内容已标注「待验证」「需确认」或「未核实」
- [ ] 对超出当前深度档范围的分析已注明限制

---

## 使用方式

1. 产出型技能在开始起草时，先加载本文件
2. 起草完成后，逐条核对并记录结果
3. 如有任何一条「否」，必须在交付物中显式说明原因，不得静默过关
4. 核对结果写入工作记录（如 `_work/claims-evidence.md` 或 `research-log.md`）

## 反例速查

| ❌ 禁止 | ✅ 正确做法 |
|--------|------------|
| "已有大量研究表明……" （无引用） | "我们在[检索范围]内找到了 X 篇相关工作，其中[具体引用]表明……" |
| "该方法显著优于基线"（无数据） | "该论文报告在[数据集]上达到[数值]，但未经我们独立验证" |
| "已下载全文并精读"（实际未获取） | "该论文为付费内容，已停在获取步骤。请将 PDF 放入 pdfs/user/ 后继续。" |
| "经分析，你的实验已成功"（仅看代码） | "代码中包含训练脚本和日志路径，但未读取日志内容。是否需要我进一步检查日志？" |
