### 1.2 这篇论文在解决什么

当时已有的预训练语言模型常按从左到右的顺序读文本。这样很适合“预测下一个词”，却限制了模型在预训练时同时利用一个词左右两边的信息。BERT 想解决的是：能否先在大量无标注文本上学到深层双向表示，再用几乎同一套模型快速适配分类、推断、问答和序列标注。

作者强调，这个限制对问答等 token-level 任务更明显，因为判断某个位置时往往需要同时看问题和完整上下文。（§1，PDF pp.1–2 / 4171–4172）

### 1.3 核心点子

BERT 没有发明新的 Transformer 主体。它使用标准 Transformer encoder，把部分输入词遮住后再恢复它们，以避免目标词直接“看见自己”；同时加入句间关系判断。预训练结束后，只增加很小的任务输出层，并对全部参数做 fine-tuning。（§3、§3.1，PDF pp.3–5 / 4173–4175）

可以把它理解为“先做带遮挡的双向完形填空，再针对具体考试微调”。类比的边界是：MLM 只预测被选中的 token，不是自然文本生成；NSP 也只是二分类代理任务，不等于完整篇章理解。

### 1.4 最强证据与最大疑点

| 主张 | 来源位置 | 证据类型 | 支持状态 | 限制 |
|---|---|---|---|---|
| 双向 MLM 整体优于 left-to-right 预训练 | §5.1、Table 5，PDF p.8 / 4178 | 对照消融 | 支持 | LTR 对照同时改变目标与 attention constraint |
| NSP 在本文设置中有帮助 | §5.1、Table 5 | 去除组件消融 | 部分支持 | 不同任务降幅差异很大 |
| 扩大模型规模提升下游结果 | §5.2、Table 6，PDF p.9 / 4179 | 多规模实验 | 支持趋势 | L/H/A 并非每次只改变一个变量 |
| BERT 在多类 NLU 任务上有效 | §4、Tables 1–4 | 多基准结果 | 支持 | headline 数字包含口径、集成或外部数据条件 |

### 1.5 是否值得细读

值得。优先顺序是：§3.1 的 MLM/NSP → §5 的消融 → §4 的指标口径 → Appendix A/C 的训练与复现细节。只读摘要会错过两个关键限定：GLUE 的 80.5 与 Table 1 的 82.1 不是同一口径；SQuAD v1.1 的 93.2 也不是裸单模型结果。

### 2.2 方法与数据流主线

#### 2.2.1 输入表示

BERT 使用 30,000 词的 WordPiece vocabulary。序列以 `[CLS]` 开头，两段文本用 `[SEP]` 分隔，并用 A/B segment embedding 区分。第 $i$ 个位置的输入为三种向量之和：

$$
e_i = e_i^{token} + e_i^{segment} + e_i^{position}.
$$

`[CLS]` 的最终向量用于句级分类；token 向量用于序列标注或问答。（§3、Figure 2，PDF pp.4–5 / 4174–4175）

#### 2.2.2 模型规模

- BERT_BASE：$L=12$、$H=768$、$A=12$，约 110M 参数。
- BERT_LARGE：$L=24$、$H=1024$、$A=16$，约 340M 参数。
- feed-forward/filter size 统一为 $4H$。

BASE 被刻意设置为与 OpenAI GPT 尺寸接近；作者认为关键结构差异是 BERT 的双向 self-attention 与 GPT 的 left-only attention。（§3，PDF p.3 / 4173）

#### 2.2.3 Masked Language Model

每个序列随机选择 15% 的 WordPiece 位置集合 $M$。对每个被选位置，80% 换成 `[MASK]`，10% 换成随机 token，10% 保持原 token，然后只预测这些位置：

$$
\mathcal{L}_{MLM} = -\sum_{i\in M}\log p(x_i\mid \tilde{x}).
$$

混合替换用于缓解 `[MASK]` 只在预训练出现、微调时不出现的 mismatch。作者同时承认 MLM 每批只监督 15% token，收敛比 LTR 略慢。（§3.1，PDF p.4 / 4174；Appendix A.1/C.1）

#### 2.2.4 Next Sentence Prediction

一半样本的 B 是 A 的真实后续（`IsNext`），另一半从语料随机抽取（`NotNext`），使用 `[CLS]` 向量做二分类。总预训练目标是 MLM 与 NSP 平均损失之和。（§3.1；Appendix A.2，PDF p.13 / 4183）

作者脚注明确提醒：未经下游微调的 `[CLS]` 向量并不是天然通用的 sentence representation。（§3.1 脚注 6）

#### 2.2.5 Fine-tuning

预训练参数初始化下游模型，通常只增加一个输出层，然后端到端更新全部参数。单句、句对、问答和 token-level 任务通过改变输入打包与输出层共享同一骨干。（§3.2、Figure 1）

### 2.3 预训练配置与成本

| 项目 | 论文设置 | 来源 |
|---|---|---|
| 语料 | BooksCorpus 800M + Wikipedia 2,500M words | §3.1，PDF p.5 / 4175 |
| 训练 | batch 256 sequences，1M steps，约 40 epochs | Appendix A.2，PDF p.13 / 4183 |
| 优化 | Adam，LR 1e-4，warmup 10k，dropout 0.1，GELU | Appendix A.2 |
| 长度 | 90% steps 用 128，最后 10% 用 512 | Appendix A.2 |
| BASE 硬件 | 4 个 Cloud TPU（16 chips），约 4 天 | Appendix A.2 |

### 2.4 实验证据审查

#### 2.4.1 关键结果与口径

| 任务 | 论文结果 | 来源位置 | 审查结论 |
|---|---|---|---|
| GLUE | official 80.5；Table 1 Average 82.1 | §4.1、Table 1 | 两者口径不同；后者排除 WNLI |
| MNLI | LARGE 86.7/85.9 | Table 1 | m/mm 两个测试分布 |
| SQuAD v1.1 | Test F1 93.2 | §4.2、Table 2 | 7-model ensemble + TriviaQA，不是裸单模型 |
| SQuAD v1.1 | single + TriviaQA Test F1 91.8 | Table 2 | 更接近单模型口径，仍使用外部数据 |
| SQuAD v2.0 | LARGE single Test EM/F1 80.0/83.1 | §4.3、Table 3 | 未使用 TriviaQA |
| SWAG | LARGE Test accuracy 86.3 | §4.4、Table 4 | human expert 85.0 只基于 100 个样本 |

GLUE 中 BERT_LARGE 在小数据集上有时 fine-tuning 不稳定。作者做多次 random restart 并按 Dev 选最佳，因此小数据集结果包含选择敏感性；论文也说明每个 BASE/LARGE 只向 test server 提交一次。（§4.1、脚注 9）

#### 2.4.2 预训练目标消融

Table 5 的 Dev 结果如下：

| 设置 | MNLI | QNLI | MRPC | SST-2 | SQuAD F1 |
|---|---:|---:|---:|---:|---:|
| BERT_BASE | 84.4 | 88.4 | 86.7 | 92.7 | 88.5 |
| No NSP | 83.9 | 84.9 | 86.5 | 92.6 | 87.9 |
| LTR & No NSP | 82.1 | 84.3 | 77.5 | 92.1 | 77.8 |

这支持“在本文实验设置中 NSP 有帮助”，尤其 QNLI；也支持双向 MLM 整体优于 LTR。**据模型分析：** LTR 对照同时改变了训练目标和 attention constraint，所以不能把差值全部解释为“纯双向性”的因果效应。（§5.1、Table 5，PDF p.8 / 4178）

#### 2.4.3 规模与 masking 消融

Table 6 从 `3/768/12` 扩展到 `24/1024/16` 时，MLM perplexity 从 5.84 降到 3.23，MNLI 从 77.9 升到 86.6，MRPC 从 79.8 升到 87.8，SST-2 从 88.4 升到 93.7。

**据模型分析：** 这证明总体 scaling trend，但 L、H、A 并非每行只改变一个变量，不能独立归因到“层数”或“头数”。（§5.2、Table 6，PDF p.9 / 4179）

Appendix C.1 报告 1M steps 相比 500k steps在 MNLI 上再增加接近 1 point；MLM 略慢收敛，但绝对准确率很早就超过 LTR。Table 8 中标准 80/10/10 的 MNLI/NER fine-tune/NER feature 为 84.2/95.4/94.9；它不是每个单项的最高值，更合理的说法是跨 fine-tuning 与 feature-based 使用较均衡。

### 2.5 局限与批判性判断

#### 作者明确写出的边界

- `[MASK]` 造成 pre-training/fine-tuning mismatch。（§3.1）
- MLM 只预测 15% token，收敛略慢。（Appendix A.1/C.1）
- 长序列 attention 成本二次增长。（Appendix A.2）
- BERT_LARGE 在小 GLUE 数据集上微调不稳定，需要 random restarts。（§4.1）
- 论文没有独立 Limitations 章节，不能说作者系统讨论了公平性、偏见或能耗。

#### 据模型分析

- 论文最强证据支持的是“MLM 双向预训练 + 模型规模 + 统一微调”整体方案；若要拆分每项的纯因果贡献，Table 5/6 仍不够正交。
- headline 指标可读性弱：official/table、single/ensemble、外部数据条件若不同时报告，容易夸大可复现性。
- NSP 在本文设置中的消融有效，但本文自身证据不足以把它提升为所有预训练模型都必须采用的普遍结论。

### 2.6 复现信号

- 官方仓库提供 checkpoints 与 SQuAD、MultiNLI、MRPC 等微调入口。
- 官方仓库明确说明：公开预训练脚本不是论文实际使用的原始 C++ 训练代码，只按论文描述生成数据；这限制了“代码级完全复刻”的说法。
- 官方说明多数 BERT_LARGE 论文设置难以在 12–16GB GPU 上复现，原微调使用 64GB Cloud TPU；BERT_BASE 示例也约需至少 12GB GPU。
- 最小可行复现建议：先用官方 BERT_BASE checkpoint 复现 MRPC 或 SQuAD fine-tuning，并固定多个随机种子；不要从零重跑 1M-step 预训练作为入门第一步。
- 本例只生成阅读报告，不运行实验；若继续，应另建 `reproduction-checklist.md`。
