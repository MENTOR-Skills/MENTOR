## 后续方向（优先）

| 方向 | 做什么 | 验收直觉 |
|------|--------|----------|
| **F1 Agent 盲测补全** | 无内容包场景：用纯模型生成入门材料（全部标注 `model-knowledge`）；空搜索结果降级：停车问用户是否接受「全部需自行核实」 | Agent 在无 embbed-ai 内容包时仍产出五件套；全部渠道空时不静默跳过 |
| **搜索可重复性加强** | 论文渠道优先走 F2 `search_arxiv.py` + `search_openalex.py`，课程/开源渠道固化搜索关键词模板 | 两次相同主题搜索的结果重叠率 ≥50% |
| **领域内容包补全** | `_template` 补 pack.yaml / curriculum.md / glossary.md 模板；至少再补 1 个方向（如 CV/NLP 入门） | 新建领域包时可照模板复制即用 |
| **与 F2 衔接自动化** | F1 `starter-resources.md` 中的论文自动作为 F2 检索种子；`glossary.md` 的术语自动注入 F2 搜索 query | 学生在 F1 完成后说「带这些论文进 F2 做全景」，Agent 自动读取五件套 |

## 演示话术（再现步骤）

```
1. 用户说：
   「我对具身智能完全不了解，学过 ML 入门和 Python，每周约 10 小时，想系统入门」

2. Agent 进入 F1 → domain-onboarding：
   阶段一：确认背景（大三，ML+编程 OK，3D 视觉和 RL 未学，偏操纵方向）
   阶段二：调用 domain-resource-search → 3 渠道并行搜索 → _work/f1-search-results.md
   阶段三：基于搜索结果依次生成
     - glossary.md（18 条术语，来源标注 + 阅读顺序图）
     - learning-map.md（4 阶段 80-120h，对齐学生背景）
     - prerequisite-gap.md（3 blocker / 2 建议补 / 补课顺序）
     - starter-resources.md（3 渠道分组，来源标签 + 诚信声明）
     - first-practice.md（Habitat PointNav 或 PyBullet CartPole 双选项）
   阶段四：诚信勾选 → UTF-8 编码检查 → 更新 research-log

3. 验证：
   python tests/scripts/run_f1_script_smoke.py    # 0 FAIL
   python shared/scripts/check_encoding.py tests/runs/f1-test/   # OK
```

## 设计决策记录

| 决策 | 结论 | 日期 |
|------|------|------|
| 单技能 vs 双技能 | **双技能**：domain-onboarding（主流程）+ domain-resource-search（内部执行）；职责清晰、可独立测试 | 2026-07-19 |
| 搜索与框架的顺序 | **先搜索、再框架**：搜索步骤 2 完成前不产出任何教学材料；术语/阶段/资源可追溯 | 2026-07-19 |
| 搜索渠道数 | **3 渠道直接并行**（不再让用户勾选 8 个渠道）；论文 / 学习资源 / 开源实践 | 2026-07-19 |
