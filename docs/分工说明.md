## 2. 各角色本周交付

### 甲 · F1 入门

| 项 | 内容 |
|----|------|
| 技能 | `skills/domain-onboarding/`（新建） |
| 交付物 | 能产出或能按模板填写：`learning-map.md` 等五件（见 `docs/接口与协议.md`） |
| 内容包 | 可选：在 `domain-packs/` 放一个极薄示范；无包时须能靠对话生成 |
| 依赖 | 需要文献列表时：调用 F2 侧检索技能，或本周降级为「用户给定链接 + 内容包」 |
| 完成观感 | 给定本科背景与目标方向 → 得到个性化入门五件套，而不是一篇空泛领域介绍 |

### 乙 · F2 文献全景

| 项 | 内容 |
|----|------|
| 起点 | 现有 `demo_skills/literature-survey/`（检索、精读入库、核对、可视化、原综述写作、原总控） |
| 动作 | 迁入或复制到本仓库 `skills/` 后改造；**完成标准改为**全景说明 + 精读清单 + 关系图，不强制长综述 |
| 主交付 | `literature-landscape.md`、`reading-shortlist.md`、`references.json`、`viz/index.html` |
| 重点 | 付费墙须能停下写 `WAITING_*.md`；关系若为推断须标注 |
| 完成观感 | 给定主题 → 有可核对的精读清单与全景说明，关系图数据来自文献库 |

### 丙 · F3 单篇深读

| 项 | 内容 |
|----|------|
| 技能 | `skills/paper-deep-read/`（独立入口）；精写入库可复用/调用 `literature-reader` |
| 深度 | 快速判断 / 深入分析 / 复现条件 三档（档位细则待协议确认后写死） |
| 主交付 | `reading-reports/<id>.md`；复现档附 `reproduction-checklist.md`；**回写同一 `references.json`** |
| 完成观感 | 给定一篇 PDF → 有人读报告，且文献库中出现对应条目 |

### 丁 · 横切 + F5 写作

| 项 | 内容 |
|----|------|
| 横切 | `campus-research-orchestrator`（路由骨架）、`research-workspace` 说明、`academic-honesty`、`AGENTS.md`、`shared/` 共用稿 |
| 写作 | 强制 `_work/claims-evidence.md`；`meeting-one-pager.md` / `qna-prep.md`；可选 `stage-writeup.md` |
| 可选 | `adversarial-lite` 薄版（只输出挑刺意见，不改终稿） |
| 完成观感 | 用户一句话能被路由到 F1/F2/F3/F5；组会材料能挂到本地或文献证据路径 |

## 5. 后续重组（预先约定）

若 F1/F3 与 F2 都用到检索或精读：

1. 本周允许在各自 `SKILL.md` 中写明调用现有技能，或暂时重复步骤说明并标注 `TODO: extract shared`  
2. 接口稳定后，单开 PR 用 `git mv` 抽到共享技能目录，再改各处引用  

此流程在 Git 上可行，不阻塞本周并行。
