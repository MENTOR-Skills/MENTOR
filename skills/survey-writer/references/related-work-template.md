# Related Work 短文模板（论文章节用）

按**主题**组织（IEEE/ACM Related Work 常见做法），不要按发表年份流水账。  
每主题段落建议：概括该线工作 → 局限 → 与本稿关注点的差异。

```markdown
## Related Work

### <Theme 1>
简述该线如何定义问题与主流做法，引用 2–4 篇代表作（含近期工作）。
与本稿差异：…

### <Theme 2>
…
与本稿差异：…

### Positioning
一段话说明本稿落在哪条线、补哪个缺口；避免「我们也做了 X」清单。
```

注意：

- Related Work ≠ 完整 Survey；控制在 seminal + 近期代表 + 最近邻方法。
- 只引用 `references.json` 内条目。
- 交付文件名建议：`related-work.md`（工作区根）。
