# survey.md 模板（独立文献综述）

写成可读的综述文章，不是检索实验日志。结构对齐常见 CS survey / review 习惯：主题综合 + 比较 + 开放问题（参见 `venue-writing-norms.md`）。

```markdown
# <题目>

## 摘要
150–300 字（或英文 Abstract 150–250 words）：覆盖范围、主线结论、意义。
可仿「背景 → 问题 → 本文组织方式 → 主要观察」四句式（IEEE 摘要习惯的弱化版）。

## 关键词
3–6 个（中英均可；若对标 ACM，可另附 CCS 概念，非强制）。

## 1. 引言
- 问题背景与动机
- 综述范围（主题、时间窗口、文献类型）一句话即可
- 本文结构导航
不要粘贴检索式、纳入排除表或漏斗统计。

## 2. <主题一：…>
段落综合：问题设定、方法演进、共识与分歧；点名代表工作（用库内 id / 作者年份）。
避免逐篇摘要堆砌。

## 3. <主题二：…>
…

## 4. <主题三：…>
…（至少 3 个主题节）

## 5. 比较与讨论
可用表格对比设定、机制、指标、开源情况；讨论不可比之处与推断性判断（标「推断」）。

## 6. 开放问题与展望
每条尽量关联前文已引用工作；区分「据当前文献库可见」与全域断言。

## 7. 结论
短段落收束主线，不引入新文献。

## 参考文献
仅列出 `references.json` 中出现的条目。
推荐两种之一（与用户确认）：
- **数字序**：正文 [1]、[2]，文末按出现序或作者序编号（IEEE 常见）
- **作者—年份**：正文（Author, year），文末按姓氏排序
```

### 篇幅建议（标准档）

- 中文约 2500+ 字，或等价英文；主题节 ≥ 3。
- 更长「投稿向」篇幅须用户明确要求后再扩展。
