# survey.md 模板（分类框架综述 · 默认写法包）

对标 ACM CSUR / 多数 CS·EE survey：**显式分类 + 主题节 + 比较 + 开放问题**。  
写成可读的综述文章，不是检索实验日志。详见 `venue-writing-norms.md` §3。

```markdown
# <题目>

## 摘要
150–300 字（或英文 Abstract 150–250 words）：覆盖范围、分类维度、主线结论、意义。
可仿「背景 → 问题 → 本文如何组织主题 → 主要观察」。

## 关键词
3–6 个（中英均可；对标 ACM 时可另附 CCS 概念占位，非强制）。

## 1. 引言
- 问题背景与动机
- 综述范围（主题、时间窗口、文献类型）— 一句话即可
- **分类维度 / taxonomy** 一句话预告（本写法包核心）
- 本文结构导航
不要粘贴检索式、纳入排除表或漏斗统计（系统梳理模式改用 `systematic-mapping-template.md`）。

## 2. 背景与预备知识（可选）
统一术语、问题形式化、评价指标约定。

## 3. <主题一 / 分类支路 A：…>
段落综合：问题设定、代表方法、共识与分歧；点名库内代表作。
避免逐篇摘要堆砌。

## 4. <主题二 / 分类支路 B：…>
…

## 5. <主题三 / 分类支路 C：…>
…（至少 3 个主题节）

## 6. 比较与讨论
表格对比设定、机制、指标、开源/复现情况；讨论不可比之处与推断性判断（标「推断」）。

## 7. 开放问题与展望
每条尽量关联前文已引用工作；区分「据当前文献库可见」与全域断言。

## 8. 结论
短段落收束主线，不引入新文献。

## 参考文献
仅列出 `references.json` 中出现的条目。
推荐两种之一（与用户确认，全文统一）：
- **数字序**：正文 [1]、[2]（IEEE 常见）
- **作者—年份**：正文（Author, year），文末按姓氏排序
预印本须标 `(preprint)`。
```

### 篇幅与语料

- 中文约 **2500+ 字**；主题节 ≥ 3。
- 建议 ≥ **8** 篇有 `reading_notes`；不足须在摘要/引言声明局限。
- 更长「投稿向」篇幅须用户明确要求后再扩展。
