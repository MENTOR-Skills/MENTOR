## 诚信声明

### 资源来源统计

| 标签 | 数量 | 含义 |
|------|------|------|
| `search-verified` | N | Agent 通过检索管线验证过存在性（论文/数据集） |
| `community-curated` | M | 来自 GitHub awesome-list 和社区讨论，经过人工筛选 |
| `model-suggested, MUST verify` | K | 模型推荐（课程/教材/教程），**使用前必须自行核实** |

### 重要提示

- **`model-suggested` 资源未经 Agent 独立验证。** 链接可能失效、内容质量参差。请在阅读前自行评估。
- **论文标题已在 arXiv/OpenAlex 上抽查验证**（抽查前 3 篇）。如发现标题不匹配，请告知以修正。
- **3 渠道默认全跑**（论文 / 学习资源 / 开源实践），无需用户勾选。
- **本材料聚焦入门。** 深入学习后，建议进入 F2 文献全景进行系统检索。

## 填写指南（给 Agent）

1. **3 渠道全部生成**——不再有「若未被勾选则省略」逻辑；每个渠道至少 2 条
2. **论文优先列综述**——综述比单篇更适合入门
3. **数量控制（标准档）：**
   - 论文：8–15 篇（综述占 30–50%）
   - 学习资源：5–8 项（课程 1–3 + 教材 1–2 + 教程 1–3 + 社区 1–2）
   - 开源与实战：3–6 个（GitHub 仓库 + 数据集/基准）
4. **每条资源信息必须完整**——标题、链接、一句话说明、来源标签缺一不可
5. **诚信声明不可省略**——这是 F1 的安全底线
