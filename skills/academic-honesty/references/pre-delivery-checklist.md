# 补充检查项（按功能）

> 在 `shared/honesty-checklist.md` 五类 A–E 基础上，各 F1–F5 功能的额外检查点。
> 由 `academic-honesty` 在识别调用方技能后加载对应小节。

## F1 · 领域入门

- [ ] 学习资源推荐是否真实存在（非编造的教材名、课程链接、论文标题）
- [ ] 术语和学习阶段是否可追溯到搜索结果或 domain-pack（非纯模型凭空生成）
- [ ] 「必读经典」清单是否有依据（来自搜索结果 / domain-pack / 模型建议并已标注）
- [ ] 术语表定义是否优先以搜索结果为准；domain-pack 作为补充验证；纯模型生成须标注
- [ ] 首次练习的可操作性：是否具体到「做什么、用什么、检验标准是什么」
- [ ] 前提知识缺口是否基于用户自述背景（非模型猜测）
- [ ] `starter-resources.md` 是否按 3 渠道分组（论文/学习资源/开源与实践），来源标签是否正确

## F2 · 文献全景

- [ ] `references.json` 中每条是否可追溯到检索结果或用户提供
- [ ] `relations` 中 `basis: inferred` 的边是否在正文中标注「推断」
- [ ] 全景说明中的「代表工作」是否确实在文献库中有 `reading_notes`
- [ ] 精读清单推荐理由是否基于文献库实际内容（非编造）
- [ ] 可视化数据是否全部来自 `references.json`（无图中编造节点/边）

## F3 · 单篇深读

- [ ] `reading_notes` 四字段（problem/method/results/limitations）是否基于实际阅读
- [ ] 「复现条件」是否逐一可核对（非「需要 GPU」这种空泛描述）
- [ ] 回写 `references.json` 的字段是否与协议 §6 一致
- [ ] 若论文为付费且用户未补 PDF：是否标注 `abstract_only` 并声明局限

## F4 · 阶段总结

- [ ]「已完成」每一条是否挂本地路径、commit 或「口头进展」标签
- [ ] 是否有因「有训练脚本」错误断定为「实验已成功」的条目
- [ ] 扫描范围是否在 `scope.md` 授权路径内
- [ ] 结果摘要中的数值是否有 `metrics.csv` / 日志 / 结果文件可核验
- [ ] 阻塞清单是否诚实（不隐瞒已知问题）

## F5 · 学术写作

- [ ] `_work/claims-evidence.md` 是否存在且每条关键主张挂证据路径或标缺口
- [ ] 是否从零撰写了 Related Work / 长综述（应路由到 F2 `survey-writer`）
- [ ] 交付文件名是否仅 ASCII（禁止 `paper-section-方法.md`）
- [ ] PPT 图表数据是否来自可核对材料（不编造）
- [ ] 润色/改写是否保留了原意且未添加无证据声称
