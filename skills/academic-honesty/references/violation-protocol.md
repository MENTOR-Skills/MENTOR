# 违规处理协议

> 被 `academic-honesty` 在检测到勾选清单「否」时调用。
> 每条违规必须按对应类别处理，不得静默跳过。

## A 类：文献与引用违规

| 违规 | 处理 | 是否必须停车 |
|------|------|--------------|
| 引用无法核实（DOI 无效、标题不匹配） | 删除该引用及依赖它的主张；或标为「未核实」并在正文显式声明 | 是（用户确认处理方式） |
| 模型推断的关系未标注 | 补充标注「推断」或 `basis: inferred`；不得改写为事实 | 否（Agent 自行修正） |
| 未读论文标为「已精读」 | 改为实际状态（`abstract_only` / `to_read`）；已写的精读笔记若为推测则删除 | 是（涉及诚信） |
| 编造的引用（完全不存在） | **立即删除**；回溯依赖该引用的所有主张一并删除；记入 research-log.md | 是（严重违规） |

## B 类：实验与数据违规

| 违规 | 处理 | 是否必须停车 |
|------|------|--------------|
| 无日志/CSV 支撑的数值声称 | 删除数值或标「未经核对」；不得保留无证据数字 | 是 |
| 因「有训练脚本」断定「实验已成功」 | 改为「存在训练脚本，未读取日志验证」；不得写「已成功」 | 否（Agent 自行修正） |
| 编造的实验数值/曲线 | **立即删除**；记入 research-log.md | 是（严重违规） |

## C 类：下载与获取违规

| 违规 | 处理 | 是否必须停车 |
|------|------|--------------|
| 绕过付费墙获取全文 | **不得保留**该 PDF；改为 `waiting_user_pdf`；写 WAITING 请用户补 | 是 |
| 假装已获取全文（实际未获取） | 改为 `abstract_only` 或 `waiting_user_pdf`；删除基于「全文」的声称 | 是 |
| 未确认 LICENSE 就使用迁移材料 | 停下，请团队确认 LICENSE | 是 |

## D 类：隐私与授权违规

| 违规 | 处理 | 是否必须停车 |
|------|------|--------------|
| 扫描了 scope.md 未授权的路径 | **立即停止**；从交付中删除来自该路径的内容；记入 research-log.md | 是 |
| 交付中含有密钥/令牌/`.env` 内容 | **立即删除**敏感内容；重新生成交付文件；记入 research-log.md | 是（安全红线） |
| 用户本地材料被外传 | **立即停止**；通知用户；记入 research-log.md | 是（安全红线） |

## E 类：不确定处未声明

| 违规 | 处理 | 是否必须停车 |
|------|------|--------------|
| 不确定的结论写成了事实 | 改为「推断」「据模型分析」「待验证」等标注 | 否（Agent 自行修正） |
| 超出深度档范围的分析未标局限 | 补充深度档局限声明（如「标准档，未做系统对比」） | 否（Agent 自行修正） |

## 通用规则

- A–D 类中标注「是（严重违规）」的条目：**必须停车**等用户确认，不得自行决定通过
- 任何违规处理后，必须重新跑对应类别的勾选检查
- 所有违规及处理结果写入 `research-log.md`
- 同一文件多次违规：逐条处理，不批量静默
